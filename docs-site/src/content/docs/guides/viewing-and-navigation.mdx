## Responsive navigation

On narrower terminals Hike can switch to a single-pane layout automatically.
The practical movement model is:

- <kbd>Ctrl</kbd>+<kbd>N</kbd> jumps to the active sidebar view
- <kbd>Ctrl</kbd>+<kbd>G</kbd> returns to the Markdown content
- <kbd>F2</kbd> still toggles navigation visibility

When the terminal grows wide enough again, Hike restores the normal split view.

## Local browser behavior

The local browser now supports two modes:

- **tree** — the original hierarchical directory tree
- **flat list** — a relative-path list optimized for quick scanning

Use <kbd>Ctrl</kbd>+<kbd>Shift</kbd>+<kbd>L</kbd> or `m` while the local browser
has focus to toggle modes.

In flat-list mode:

- selecting a directory changes the local browser root
- empty directories without visible Markdown descendants are hidden
- `../` appears when a parent directory is available
- <kbd>Backspace</kbd> moves to the parent directory directly

## Startup selection

When Hike starts from a directory, it can auto-open a preferred file and select
it in the sidebar. The default order is:

1. `INDEX.md`
2. `README.md`
3. the first visible Markdown file in local-browser order

You can override that order in the configuration file. See
[Configuration → File Browser & Startup](../configuration/file-browser-and-startup/).
